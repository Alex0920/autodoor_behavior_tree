# 多屏兼容逻辑重复分析与重构建议

## 文档信息
- **版本**: V1.0
- **日期**: 2026-05-10
- **分析范围**: 全项目中所有涉及虚拟屏幕边界计算的代码
- **根因**: `magnifier.py` 有完善的 `_get_virtual_screen_bounds()` 私有方法，但未被复用，导致其他模块各自重复实现

---

## 1. 问题总览

### 1.1 核心问题

项目中存在 **9 处** `screeninfo.get_monitors()` 调用点，每处都独立计算虚拟屏幕边界（min_x, min_y, max_x, max_y），逻辑高度雷同但实现分散，且 fallback 策略不统一。

### 1.2 影响范围

| 文件 | 调用次数 | 用途 |
|------|----------|------|
| `bt_utils/magnifier.py` | 1 | 放大镜窗口定位（最完善，含三级 fallback） |
| `bt_gui/bt_editor/property.py` | 5 | RegionField / PositionField / OffsetField / ScreenshotField / PreviewContext |
| `bt_gui/widgets.py` | 2 | 颜色选择器窗口大小 + 坐标转换 |
| `bt_gui/script_tab.py` | 1 | 脚本坐标选择 |

---

## 2. 逐处详细分析

### 2.1 唯一的"规范实现" — `magnifier.py` (行 38-79)

```python
def _get_virtual_screen_bounds(self) -> Tuple[int, int, int, int]:
    # 三级 fallback 策略：
    # 1. screeninfo 库（最精确）
    # 2. Win32 API GetSystemMetrics(76-79)（次精确）
    # 3. tkinter winfo_screenwidth/height（兜底）
```

**特点**:
- ✅ 三级 fallback 策略，健壮性最强
- ✅ 结果缓存到 `self._virtual_screen_bounds`，避免重复计算
- ✅ 返回标准化的 `(min_x, min_y, max_x, max_y)` 四元组
- ❌ 是**私有方法**（`_` 前缀），外部模块无法导入复用
- ❌ 依赖 `self.window`（用于第三级 fallback），非静态方法

## 3. 差异对比矩阵

### 3.1 核心计算逻辑对比

| 位置 | 取 min_x | 取 min_y | 取 max_x | 取 max_y | fallback 策略 |
|------|:--------:|:--------:|:--------:|:--------:|---------------|
| magnifier.py:38 | ✅ | ✅ | ✅ | ✅ | screeninfo → Win32 API → winfo (三级) |
| property.py:442 | ✅ | ✅ | ✅ | ✅ | 无 |
| property.py:1010 | ✅ | ✅ | ✅ | ✅ | 无 |
| property.py:1416 | ✅ | ✅ | ✅ | ✅ | 无 |
| property.py:1580 | ✅ | ✅ | ✅ | ✅ | 无 |
| property.py:2531 | ✅(取反) | ✅(取反) | ❌ | ❌ | except → (0,0) |
| widgets.py:234 | ✅ | ✅ | ✅ | ✅ | ImportError弹窗 → Exception→1920x1080 |
| widgets.py:280 | ✅ | ✅ | ❌ | ❌ | 裸except → (0,0) |
| script_tab.py:531 | ✅ | ✅ | ✅ | ✅ | ImportError弹窗 → Exception→1920x1080 |

### 3.2 fallback 策略对比

| 策略等级 | 描述 | 使用位置 |
|----------|------|----------|
| **三级** | screeninfo → Win32 API → winfo | magnifier.py (1处) |
| **二级** | screeninfo → 固定分辨率回退 | widgets.py:234, script_tab.py:531 (2处) |
| **一级** | screeninfo → 零偏移回退 | property.py:2531, widgets.py:280 (2处) |
| **零级** | 仅 screeninfo，无 fallback | property.py:442/1010/1416/1580 (4处) |

### 3.3 关键风险

1. **零级 fallback 的 4 处**：如果 `screeninfo` 未安装或 `get_monitors()` 抛异常，会直接崩溃
2. **裸 except 的 1 处**（widgets.py:280）：吞掉所有异常，可能掩盖真实问题
3. **不一致的回退值**：有的回退到 `(0,0,1920,1080)`，有的回退到 `(0,0)`，行为不统一

## 5. 重构建议

### 5.1 提取公共工具函数

将 `magnifier.py` 中的 `_get_virtual_screen_bounds()` 提取为独立的公共模块级函数，放在 `bt_utils/helpers.py` 或新建 `bt_utils/screen_utils.py`：

```python
# bt_utils/screen_utils.py（建议新增）

from typing import Tuple, Optional
import threading

_virtual_screen_bounds_cache: Optional[Tuple[int, int, int, int]] = None
_cache_lock = threading.Lock()


def get_virtual_screen_bounds() -> Tuple[int, int, int, int]:
    """
    获取虚拟桌面边界（支持多显示器）
    
    三级 fallback 策略：
    1. screeninfo 库（最精确）
    2. Win32 API GetSystemMetrics(76-79)（次精确）
    3. 固定默认值 (0, 0, 1920, 1080)（兜底）
    
    Returns:
        Tuple[int, int, int, int]: (min_x, min_y, max_x, max_y)
    """
    global _virtual_screen_bounds_cache
    
    if _virtual_screen_bounds_cache is not None:
        return _virtual_screen_bounds_cache
    
    bounds = _compute_virtual_screen_bounds()
    
    with _cache_lock:
        _virtual_screen_bounds_cache = bounds
    
    return bounds


def get_virtual_screen_offset() -> Tuple[int, int]:
    """
    获取虚拟桌面的左上角偏移量
    
    Returns:
        Tuple[int, int]: (offset_x, offset_y)
        多屏时为负值，单屏时为 (0, 0)
    """
    min_x, min_y, _, _ = get_virtual_screen_bounds()
    return (-min_x, -min_y)


def _compute_virtual_screen_bounds() -> Tuple[int, int, int, int]:
    """实际计算虚拟屏幕边界"""
    # 策略1: screeninfo
    try:
        import screeninfo
        monitors = screeninfo.get_monitors()
        if monitors:
            return (
                min(m.x for m in monitors),
                min(m.y for m in monitors),
                max(m.x + m.width for m in monitors),
                max(m.y + m.height for m in monitors),
            )
    except ImportError:
        pass
    except Exception:
        pass
    
    # 策略2: Win32 API
    try:
        import ctypes
        user32 = ctypes.windll.user32
        vl = user32.GetSystemMetrics(76)
        vt = user32.GetSystemMetrics(77)
        vw = user32.GetSystemMetrics(78)
        vh = user32.GetSystemMetrics(79)
        if vw > 0 and vh > 0:
            return (vl, vt, vl + vw, vt + vh)
    except Exception:
        pass
    
    # 策略3: 默认值
    return (0, 0, 1920, 1080)
```

## 6. 附加发现：`ImageGrab.grab(all_screens=True)` 分散调用

除了虚拟屏幕边界计算外，`ImageGrab.grab(all_screens=True)` 也存在类似分散问题：

| 文件 | 行号 | 场景 |
|------|------|------|
| bt_utils/screenshot.py | 23, 34 | ScreenshotManager（规范实现） |
| bt_utils/magnifier.py | 166 | 放大镜截图 |
| bt_gui/widgets.py | 276 | 颜色拾取截图 |
| bt_gui/bt_editor/property.py | 2474, 2476, 2485 | PreviewContext 截图 |

建议：所有截图操作统一通过 `ScreenshotManager` 或 `ExecutionContext.get_screenshot()` 进行，避免直接调用 `ImageGrab.grab()`。

## 7. 总结

| 维度 | 现状 | 重构后 |
|------|------|--------|
| 重复实现数 | 9 处 | 1 处（公共函数） |
| fallback 策略 | 4 种不同等级 | 统一三级 fallback |
| 代码行数 | ~60 行分散 | ~40 行集中 |
| 新增模块风险 | 修改任一处需检查所有调用点 | 只需修改公共函数 |
| screeninfo 不可用时 | 4 处直接崩溃 | 全部优雅降级 |
